# ERP System Release Note_2026 07 3주차

기간: 2026-07-13 ~ 2026-07-17

## 요약

이번 주에는 AI 업무비서가 신규 메뉴로 추가되고, 내부 MCP 도구를 통해 타임시트/프로젝트/의견청취/운영 요약을 자연어로 조회할 수 있도록 확장되었습니다. 전사일정은 Google Calendar 연동과 휴가 일정 표시 오류 대응이 보강되었고, 타임시트는 프로젝트 구분, 업무 내용, 작업유형 다중 선택, 원가구분 자동 적용을 중심으로 입력 구조가 크게 정리되었습니다.

프로젝트 관리는 계약금액/매출원가 세부 금액을 DB 컬럼으로 분리하고 PM 수정 권한을 사번 기준으로 보강했습니다. 의견청취와 견적관리는 작성자/관리자 권한, 읽기전용 상세, 답변 수정/삭제 등 실사용 권한 흐름을 보완했습니다. 공통 UI에서는 모달/드로어 외부 클릭 및 Escape 닫힘 동작을 입력 취소 확인 팝업 중심으로 정리했습니다.

## 메뉴별 변경 사항

### AI 업무비서

#### 기능 수정 사항

- 신규 `AI 업무비서` 메뉴가 추가되어 내부 MCP 도구 기반 자연어 조회가 가능해졌습니다.
- 타임시트, 프로젝트, 의견청취, 운영 요약을 대화형으로 조회할 수 있도록 도구 목록, 채팅 API, 프론트 화면이 추가되었습니다.
- 타임시트 질문에서 `지난주`, `이번달`, `지난달`, 기간 범위, 작성/미작성 인원, 작성중 상태 등 자연어 표현 인식이 보강되었습니다.
- 프로젝트 조회에서 `최근`, `현황`, `보여줘` 같은 업무 지시어가 검색어로 남아 결과가 0건이 되는 문제를 수정했습니다.
- 프로젝트와 의견청취, 운영 요약의 기본 조회 제한을 완화해 사용자가 요청한 전체 목록을 더 자연스럽게 확인할 수 있도록 했습니다.
- AI 응답 화면에서 내부 MCP 호출명과 arguments가 사용자에게 노출되지 않도록 UI 표시와 메시지 상태 저장을 정리했습니다.

#### 기술적인 내용

- `backend/app/routers/ai.py`, `backend/app/mcp/tools.py`가 추가되어 `/api/ai/*`, `/api/mcp*` 기반 내부 도구 호출 구조가 구성되었습니다.
- 프론트 API에 `aiApi`가 추가되고, `frontend/src/views/AiAssistantView.vue`, 라우터, 권한 유틸, 메뉴가 연결되었습니다.
- 프로젝트 상태값 `진행`, `진행 중`, `종료/완료됨` 등 legacy 표현을 표준 상태로 정규화하고, 운영 요약/경영 분석/MCP 검색의 진행중 필터가 alias를 포함하도록 수정되었습니다.
- Alembic `20260714_0006_normalize_project_status.py`가 추가되어 기존 DB 프로젝트 상태값 정규화가 필요합니다.
- Google Calendar 관련 credential/env 파일은 git 추적에서 제거되고 `.gitignore`에 반영되었습니다.

### 전사일정

#### 기능 수정 사항

- 전사일정 화면에서 Google Calendar 기반 월간 일정과 휴가 일정을 함께 표시하도록 보강했습니다.
- 전사일정 진입 또는 조회 실패 시 단순 실패 메시지가 아니라 백엔드 상세 오류를 포함해 원인 파악이 가능하도록 개선했습니다.
- 휴가 일정 API와 월간 일정 API 호출 흐름을 조정해 중복 오류 토스트가 발생하던 UX를 완화했습니다.

#### 기술적인 내용

- `backend/app/routers/schedule.py`에서 Google Calendar credentials 경로의 slash/backslash를 OS 경로로 정규화했습니다.
- `frontend/src/views/CalendarView.vue`에서 월간 일정과 휴가 일정 병합 표시 및 오류 메시지 처리가 보강되었습니다.
- 운영 환경에서는 Google Calendar 환경변수, 키 파일 경로, API 권한 설정을 재확인해야 합니다.

### 타임시트

#### 기능 수정 사항

- 타임시트 테이블 구조가 `구분 / 프로젝트 / 업무 내용 / 원가 구분 / 작업유형` 중심으로 개편되었습니다.
- 기존 SPG 화면 컬럼은 제거되고, 프로젝트 자동완성에 실행 프로젝트와 영업관리 주차/직전 주차 프로젝트가 함께 표시됩니다.
- 직접 입력 프로젝트는 `공통` 구분으로 처리되며, 연차 선택 시 프로젝트와 작업유형이 연차 기준으로 자동 보정됩니다.
- 하단 버튼명이 `행 추가`에서 `일정 추가`로 변경되고, 주간 메모 입력 박스가 제거되었습니다.
- 업무 내용 컬럼이 추가되어 기존 `notes` 필드로 저장/조회됩니다.
- 작업유형 다중 선택이 지원되며 최대 3개까지 선택할 수 있도록 제한되었습니다.
- 작업유형에 `영업 > SHOP작업`이 추가되었습니다.
- 타임시트 원가구분은 사용자가 화면에서 임의 변경하지 않고, 사용자관리의 원가구분 설정을 기준으로 서버에서 강제 적용되도록 변경되었습니다.
- 타임시트 테이블 컬럼 폭을 조정해 프로젝트명과 업무 내용 표시 가독성을 개선했습니다.

#### 기술적인 내용

- `timesheet_entries.project_source` 컬럼이 추가되어 실행/영업/공통 구분을 저장합니다.
- `users.labor_type` 컬럼이 추가되어 사용자별 원가/판관 기본값을 관리합니다.
- `backend/app/routers/timesheet.py`에서 저장 시 사용자 계정의 `labor_type`을 기준으로 원가구분을 강제 적용합니다.
- Alembic `20260714_0009_timesheet_project_source.py`, `20260714_0010_user_labor_type.py`, `20260714_0011_expand_timesheet_work_type.py`가 추가되었습니다.
- `frontend/src/views/TimesheetView.vue`에서 자동완성, 다중 작업유형, 업무 내용 입력, 컬럼 폭이 반영되었습니다.

### 프로젝트

#### 기능 수정 사항

- 프로젝트 등록/수정의 계약금액/매출원가 세부 금액이 개별 DB 컬럼으로 저장되도록 변경되었습니다.
- 기존 비고에 저장되던 프로젝트 메타 JSON은 비고 목적에 맞게 분리되고, 기존 데이터도 새 컬럼으로 이관할 수 있도록 보강되었습니다.
- 프로젝트 담당 PM은 역할 권한과 별개로 본인 담당 프로젝트를 수정할 수 있으며, 비교 기준이 이름에서 사번으로 변경되었습니다.
- 프로젝트 담당 PM 입력이 직원 선택 방식으로 변경되어 PM 사번이 함께 저장됩니다.
- 프로젝트 등록/수정 모달 외부 클릭 시 입력 취소 확인 팝업이 표시되고, X/취소 버튼은 즉시 닫히도록 동작을 분리했습니다.
- 프로젝트 목록, 매출, 매입 테이블의 페이지 크기 선택 상태 유지와 페이지 범위 보정이 추가되었습니다.
- 프로젝트 삭제 시 `project_id` 컬럼이 없는 보조 테이블까지 무조건 연결 해제하려다 발생하던 삭제 오류를 수정했습니다.

#### 기술적인 내용

- `projects` 모델에 계약금액/매출원가 세부 컬럼과 `pm_employee_code`가 추가되었습니다.
- `backend/app/utils/authorization.py`에 PM 수정 권한 판정 로직이 보강되었습니다.
- `backend/app/routers/execution.py`에서 프로젝트 dict 응답, 등록/수정 payload, 삭제 시 참조 해제 로직이 수정되었습니다.
- Alembic `20260714_0007_project_cost_detail_columns.py`, `20260714_0008_project_pm_employee_code.py`가 추가되었습니다.
- 기존 notes JSON 금액은 마이그레이션으로 새 컬럼 이관이 필요합니다.
